# audit theory implementation coverage

## Goal

审查 `doc/理论实现对应.md`、`doc/0415-00记忆认知架构.md` 与当前 Rust 代码实现之间的一致性，识别代码中的占位符/空实现/仅骨架接口，并判断 0415 架构文档中声明的核心能力哪些已经实现、哪些仍未实现或仅部分实现；在确认偏差后，校正文档并修复能在本任务范围内收敛的代码偏差。

## What I already know

* 用户先要求审查，随后明确要求“校正后，修复代码”。
* 仓库已有较完整的 cognition、memory、search、agent、ingest、CLI 模块与大量测试。
* `doc/理论实现对应.md` 当前是未提交新文件，需要和现有实现一起核对可信度。
* 当前工作树有多处未提交改动，审查时不能误把脏变更当作本轮新增实现。

## Assumptions (temporary)

* 本轮需要同时产出审查结论，并对确认属实的文档/代码偏差进行修复。
* “占位符”包含 `todo!`、`unimplemented!`、`panic!` 式临时实现、仅返回空结果的骨架逻辑、文档宣称已实现但代码只建了数据结构/迁移未接线的情况。
* “全部实现”按文档中的核心能力、约束与链路是否可从代码和测试中证实来判断，不以命名出现即算实现。

## Open Questions

* 无阻塞问题，直接通过仓库代码与文档核查。

## Requirements (evolving)

* 识别代码库中的显式占位符与疑似占位实现。
* 对照 `doc/理论实现对应.md` 判断映射是否准确、是否夸大实现状态。
* 对照 `doc/0415-00记忆认知架构.md` 判断核心模块与链路的实现覆盖。
* 输出审查发现时优先给出高风险缺口、文件引用和判断依据。
* 校正 `doc/理论实现对应.md` 中夸大、错误或缺乏限定条件的实现状态。
* 修复至少一项与审查发现直接对应、且能在当前范围内安全完成的代码问题。

## Acceptance Criteria (evolving)

* [ ] 给出占位符/空实现审查结果
* [ ] 给出 0415 架构文档实现覆盖结论
* [ ] 给出关键证据文件与测试依据
* [ ] 明确哪些是“已实现 / 部分实现 / 未实现”
* [ ] 文档校正完成且与当前代码一致
* [ ] 至少一项对应代码偏差已修复并验证

## Definition of Done (team quality bar)

* 审查结论可回溯到具体代码/测试/文档
* 不误改用户已有业务代码
* 如无代码改动，也需保持任务上下文与审查范围记录完整

## Out of Scope (explicit)

* 本轮不补齐所有缺失能力
* 本轮不重写 `doc/0415-00记忆认知架构.md`
* 本轮不做外部资料研究

## Technical Notes

* 重点文档：`doc/理论实现对应.md`、`doc/0415-00记忆认知架构.md`
* 重点代码域：`src/cognition/`、`src/memory/`、`src/search/`、`src/agent/`、`src/interfaces/cli.rs`
