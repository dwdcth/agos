## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| config + local DB state -> command execution | Retrieval mode, backend, schema state, and lexical index state determine whether a command may proceed. |
| CLI entrypoint -> DB/service layers | If the gate is skipped here, downstream ordinary retrieval and agent-search will execute under invalid or misleading runtime conditions. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | T | `src/interfaces/cli.rs` | mitigate | Enforce one shared preflight helper before `Database::open(...)` so no operational command can tamper with or bypass runtime readiness policy independently. |
| T-06-02 | R | `src/core/doctor.rs` | mitigate | Render blocked operational paths through structured `DoctorReport` failures so operators can see exactly why execution was denied. |
| T-06-03 | E | `src/core/doctor.rs` / `src/core/status.rs` | mitigate | Preserve explicit reserved semantic-mode failures and never downgrade `embedding_only` or `hybrid` into an implicit lexical fallback. |
</threat_model>
