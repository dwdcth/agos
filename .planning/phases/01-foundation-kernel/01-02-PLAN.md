## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| application -> SQLite file | Process writes durable local state and schema into a user-controlled filesystem location |
| domain record -> SQL row | Structured metadata crosses into SQL storage and must retain integrity across serialization |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-04 | T | `migrations/0001_foundation.sql` | mitigate | Keep schema explicit and additive, with typed columns for metadata and no dynamic SQL assembled from user content. |
| T-01-05 | R | `src/memory/repository.rs` | mitigate | Persist provenance and timestamps as first-class columns/JSON so later audits can trace where a record came from. |
| T-01-06 | D | `src/core/db.rs` | mitigate | Apply migrations idempotently on open and surface schema version errors immediately rather than continuing with a half-initialized database. |
</threat_model>
