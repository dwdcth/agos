Decision trace used in this plan:
- `D-01`: Use TOML as the primary local config format.
- `D-02`: Define a typed config surface in Phase 1.
- `D-03`: Model retrieval strategy as three explicit modes: `lexical_only`, `embedding_only`, `hybrid`.
- `D-04`: Keep embedding optional in foundation work.
- `D-05`: Treat all three retrieval modes as first-class states in startup/status contracts.
- `D-06`: Reserve clean extension seams for future embedding backends without making them critical-path dependencies.
- `D-07`: Preserve later lexical-plus-embedding coexistence, with lexical remaining the primary explanation source.
- `D-08`: `lexical_only` means precise, source-aware, strongly explainable retrieval.
- `D-09`: `embedding_only` means semantic-heavy recall with weaker exact explainability.
- `D-10`: `hybrid` means mixed-mode retrieval where lexical stays primary and embedding is secondary expansion/rerank.
- `D-11`: Base storage/model contracts must already preserve source, timestamp, scope, record type, truth metadata, and provenance.
